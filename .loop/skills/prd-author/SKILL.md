# Skill: prd-author

> PRD 撰写技能。从模糊需求产出结构化 PRD。

## 触发条件

用户输入新需求，或 Regression Loop 反馈 drift 信号时触发。

## 职责

1. 读取需求输入（用户描述 / 回归反馈 / LOOP-STATE.md）
2. 澄清不明确的需求点（生成具体问题列表，不猜测）
3. 结构化为 PRD 草案，必须包含：
   - 产品定位（1 句话）
   - 用户痛点（>=3 条）
   - 核心用户旅程（>=1 条）
   - MVP 功能列表（含优先级 P0/P1）
   - 功能黑名单（>=1 条）
   - 验收标准（可测试）
   - 技术方案概要
   - 主要风险
4. 更新 docs/PRD.md 和 .loop/phases/prd-state.md

## 输出格式

按 docs/PRD.md 模板结构产出。标记状态为 draft。

## 禁止行为

- 不猜测需求——不明确时生成问题列表
- 不扩大 MVP 范围——功能黑名单必须明确
- 不写不可测试的验收标准
- 不跳过功能黑名单
- 不修改 denylist 路径下的文件

## 验收自检（交由 loop-verifier 独立验证）

- 产品定位是否 1 句话
- 用户痛点是否 >=3 条
- 核心旅程是否 >=1 条
- MVP 列表是否有优先级
- 功能黑名单是否 >=1 条
- 验收标准是否可测试
- MVP 范围与黑名单是否矛盾
