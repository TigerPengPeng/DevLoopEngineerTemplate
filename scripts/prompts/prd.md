# PRD Author Loop

## 任务
根据用户需求编写完整的产品需求文档。

## 输出
在 docs/PRD.md 中编写完整的 PRD，包含：
- 产品定位与目标用户
- 用户痛点与场景
- 核心用户旅程
- MVP 功能清单（P0/P1）
- 功能黑名单（不做什么）
- 验收标准
- 技术选型建议

## 重要约束 ⚠️
1. **不要执行任何 git 命令**（git add / git commit / git push）
2. git 操作由 orchestrator 自动完成，你只需要编写文档
3. 完成后标记 .loop/phases/prd-state.md 为 done
4. 更新 LOOP-STATE.md 的 Current phase 为 design
5. 将 docs/PRD.md 添加到 Frozen Artifacts
