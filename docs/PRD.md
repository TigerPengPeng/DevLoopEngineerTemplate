## 产品定位

为个人投资者提供自动化股票行情监听服务：对接 Futu OpenD，实时监控美股和 A 股的价格波动与均线突破，事件触发时邮件告警。

## 用户痛点

1. 美股有盘前、盘中、盘后、夜盘多个时段，投资者无法全天盯盘，容易错过关键价格信号。
2. 手动计算 MA5/13/30/55 并盯盘效率低，且容易在关键突破/跌破时刻错过操作窗口。
3. 现有行情软件的提醒功能粒度粗（仅价格提醒），不支持 MA 均线穿越检测，且无法自定义分组。
4. OpenD 网关断线后需要手动重连，长时间运行稳定性差。

## 核心用户旅程

### 旅程 1: 配置并启动监听
1. 用户在 Futu 客户端将关注的股票加入某个股票分组。
2. 用户配置 OpenD 地址、邮件 SMTP、监控参数（MA 周期、波动阈值）。
3. 启动服务，自动拉取分组内股票、订阅行情、开始监控。
4. 服务持续运行，OpenD 断线自动重连。

### 旅程 2: 收到突破告警
1. 某股票价格突破 MA13 均线。
2. 系统检测到价格从 MA13 下方穿越到上方（突破事件）。
3. 系统发送邮件，包含股票名称、当前价、MA13 值、事件类型、交易时段。
4. 冷却期（15 分钟）内同类事件不再重复发送。

### 旅程 3: 收到波动告警
1. 某股票日内波动幅度超过 ±2%（相对前收盘价）。
2. 系统检测到波动超限，生成波动告警事件。
3. 系统发送邮件通知，标注波动方向（涨/跌）和幅度。

## 功能黑名单

> 明确不做的事项，防止范围蔓延。

- 不做 Web UI / 管理后台（纯 headless 服务）
- 不做自动交易 / 下单（只监听，不操作）
- 不做历史回测功能
- 不做技术指标图形渲染
- 不做多用户 / 多账户管理（单账户）
- 不做推送渠道扩展（仅邮件，不做微信 / Telegram / Webhook）

## 验收标准

> 可测试的验收条件。

- [ ] 服务能连接到 Futu OpenD (127.0.0.1:11111)，连接状态为 READY
- [ ] 能拉取指定股票分组的股票列表
- [ ] 能订阅股票基础行情并收到实时推送
- [ ] MA5/13/30/55 计算结果正确（与手动计算一致）
- [ ] 价格突破 MA 时生成突破事件，跌破时生成跌破事件
- [ ] 日内波动超阈值时生成波动告警
- [ ] 美股盘前/盘中/盘后/夜盘四个时段能正确识别和标注
- [ ] A 股股票能正常订阅和监控
- [ ] 事件触发时邮件发送成功，内容完整
- [ ] OpenD 断线后服务自动重连，重连后恢复全部订阅
- [ ] 告警冷却期内同类事件不重复发送
- [ ] docker compose up 能一键启动服务
- [ ] 重连过程无内存泄露（旧连接资源被正确释放）

## 技术方案（概要）

> 高层技术方向，详细方案见 ARCHITECTURE.md。

- 语言: Java 17
- 框架: Spring Boot 3.3.x（非 Web，headless + actuator）
- Futu SDK: com.futunn.openapi:futu-api:9.3.5308（Maven Central）
- 邮件: Spring Boot Mail (JavaMail)
- 部署: Docker + docker-compose

## 主要风险

- Futu SDK 是回调驱动的异步模型，需要桥接到同步调用模式才能在业务层方便使用。
- SDK 连接断开后如果不正确释放旧连接资源（close + 置 null），会有内存泄露风险。
- OpenD 的订阅额度有限（免费版 100 只），大量股票订阅可能超限。
- 美股夜盘 / 盘后数据是否实时推送取决于 OpenD 配置，需验证。
- A 股交易时段与美股不同，需分别处理时段映射逻辑。
